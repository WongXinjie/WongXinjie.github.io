Python的Web框架

发表于 2016-12-04

Python中的web框架，用户基数较大的大概就是Django、Flask和tornado，实际上Python的web框架多如牛毛，甚至已经有很多不再维护，只支持Python2，而个人觉得除了历史遗留项目用Python2，新项目直接上Python3还是靠谱一点。本文就谈谈个人在项目中使用过的或者感兴趣的框架。

Django

要推举Python届Web框架卫冕之王，Django应该勉强可以算是。对于学Python web开发的新人，往往被推荐的web框架就是Django。但实际上Django相当不适合给新手入门。
Django的文档相当详细，详细到这个东西在项目中怎么实现都给你写的清清楚楚。文档详细一个好处的就是开发过程中对某一点有疑问读一读文档便可以，但文档太详细太长也有坏处，吓跑新手。
Django一开始是以一个CMS框架为目的开发，一次Django的封装度实际上是想到高的。一个新手按tutorials一步一步走，的确可以讲一个简单的项目跑起来，连项目结构都已经帮你规划好了，甚至连admin都是自带的，配置一下，三下五除二一个网站便做成了。Django大概就像你的父母，什么都想帮你考虑好，你需要这个东西，我有内置的，以用一下，ORM我内置了，template系统内置了，xxx我内置了，你在settings.py文件了设置一下就可以了！
Django这种自带电池的框架，对于工期比较紧的项目，或者是数据库驱动的项目的确是一个好框架。甚至用Class-Base View你写一个models,在views里写几行代码，一个增删查改的功能就出来了，相当省事。
但凡事有代价。个人认为，一个项目中的业务层应该和视图层分开，无论是测试还是维护都有好处，但django项目中往往更推崇直接将逻辑写在views中。django自带的ORM，使用起来的确方便，但是写出低效率查询逻辑也是很方便。另外，对于想前后端分离，自己抛弃django模板系统的项目，Django也不是特别友好。
Django的一个槽点便是性能低，可能是更框架自身的实现有关。相关文章如这篇。

Flask

毫无疑问的是，Flask是web框架中最优秀的微框架，无论是生态圈还是文档。相比于Django的一坨文件，Flask一个文件就可以跑起来（当然在实际项目中为了可维护性不要这样）,是最适合新手入门的框架。
Flask框架是基于Jinja2模板引擎和WerkzeugWSGI工具集的一个微框架，再实现一些诸如路由管理和session管理等等功能组合起来。但不可否认，Flask的确好用。
Flask只提供了最基础的功能，甚至连登录认证、表单验证等功能都不实现，全部都交给了extensions,所以用Flask开发项目的时候往往requirements.txt中有不下几十个Flask相关的依赖。
Flask很适合写一些轻量级的项目，写RESTful-api也挺适合。写一些大型项目也可以，但是对项目的组织要求能力也相应教高，很多功能可能都要自己实现，以及在繁杂的拓展中选一个靠谱的插件。
在Flask项目中，和Django中的路由是在urls.py中集中管理不同，Flask往往是分散在各个文件中，以装饰器的方式分布在每一个view-function上头，这样带来的一个坏处是路由难以管理，另外一个是一个view-function可能头上顶十几个decorators，而且这些decorators还得按照一定的顺序来写。后来自0.7开始Flask从Django借鉴了Class-Base views思路，实现了Pluggable views。
